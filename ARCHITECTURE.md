## 🧮 알고리즘 흐름도

### 심플 모드 (Simple Mode) 알고리즘

```
입력: params (coin, interval, n_streak, direction)
│
├─ 1. Context 생성 및 검증
│   └─ AnalysisContext.from_params(params)
│
├─ 2. 데이터 로드
│   ├─ load_data(context)
│   ├─ Binance API 또는 CSV
│   └─ prepare_dataframe(df)
│
├─ 3. DatetimeIndex 정규화 (필수)
│   └─ normalize_indices(df) → DatetimeIndex 보장
│
├─ 4. 패턴 탐지
│   ├─ direction == 'green' → N연속 양봉
│   └─ direction == 'red' → N연속 음봉
│
├─ 5. C1 날짜 추출 (패턴 완성 후 다음 봉)
│   ├─ target_cases.index → 패턴 완성 시점
│   └─ for idx in target_cases.index:
│       └─ c1_idx = df.index[pos + 1]  # 다음 봉(C1)
│
├─ 6. C1/C2 예측 확률 계산
│   ├─ C1이 양봉인 확률
│   ├─ C1이 음봉인 확률
│   ├─ C2가 양봉일 확률 (C1 양봉 조건)
│   └─ C2가 양봉일 확률 (C1 음봉 조건)
│
├─ 7. Wilson Score Interval 계산
│   ├─ wilson_confidence_interval(p, n)
│   └─ 95% 신뢰구간 (z = 1.96)
│
├─ 8. 전일 지표 비교 분석
│   ├─ C1이 양봉인 날의 전일 지표
│   └─ C1이 음봉인 날의 전일 지표
│
├─ 9. 시간대별 분석 (1d/3d 타임프레임)
│   ├─ C1이 양봉인 날짜만 필터링
│   ├─ 각 날짜의 4시간봉 추출
│   ├─ UTC → EST/EDT 시간대 변환 (pytz 자동 처리)
│   ├─ 6개 구간(4시간씩) 분할
│   ├─ 저점/고점 발생 구간 카운트
│   └─ valid_date_count로 확률 정규화
│
└─ 10. 결과 반환
    └─ SimpleAnalysisResult
```

### 복합 모드 (Complex Mode) 알고리즘

```
입력: params (coin, interval, complex_pattern, rsi_threshold)
│
├─ 1. Context 생성 및 검증
│   └─ AnalysisContext.from_params(params)
│
├─ 2. 데이터 로드 (Simple Mode와 동일)
│   ├─ load_data(context)
│   └─ normalize_indices(df)
│
├─ 3. 패턴 매칭
│   ├─ complex_pattern 예: [1,1,1,1,1,-1,-1] (5양-2음)
│   └─ 벡터화된 패턴 매칭 (shift 연산)
│
├─ 4. C1 날짜 추출 (패턴 완성 후 다음 봉)
│   ├─ matched_patterns.index → 패턴 완성 시점
│   └─ for idx in matched_patterns.index:
│       └─ c1_idx = df.index[pos + 1]  # 다음 봉(C1)
│
├─ 5. 지표 계산
│   ├─ RSI 계산
│   ├─ 이격도 계산: D = (C - MA20) / MA20 * 100
│   └─ 전일 지표 추출
│
├─ 6. 패턴 품질 점수 계산 (Signal Score)
│   ├─ Pullback Quality (되돌림 품질) 분석
│   │   ├─ 되돌림 비율: retracement = (rise_high - drop_end) / (rise_high - rise_start) * 100
│   │   └─ 거래량 비율: vol_ratio = vol_drop / vol_rise
│   ├─ Signal Score 계산 (최대 100점)
│   │   ├─ 가격 지지 강도 (되돌림 비율): 최대 40점
│   │   │   └─ < 30%: 40점, < 50%: 30점, < 70%: 15점, >= 70%: 0점
│   │   ├─ 거래량 건강도 (거래량 비율): 최대 30점
│   │   │   └─ < 0.7: 30점, < 1.0: 20점, < 1.5: 10점, >= 1.5: 0점
│   │   └─ 추세 강도 (RSI): 최대 30점
│   │       └─ > 70: 30점, > 65: 25점, > 60: 15점, <= 60: 0점
│   └─ 신뢰도 판단: High (>= 80), Medium (>= 60), Low (< 60)
│   └─ 위치: common.py - calculate_signal_score()
│
├─ 7. RSI/이격도 구간별 승률 분석
│   ├─ RSI: 8개 구간 [0,30,40,50,60,70,80,100]
│   ├─ 이격도: 7개 구간
│   ├─ 각 구간별 다음 봉 양봉 확률
│   ├─ Wilson Score Interval 계산
│   ├─ Binomial P-value 계산
│   └─ Bonferroni 보정 적용
│
├─ 8. Bonferroni 보정
│   ├─ α_adjusted = α / n_comparisons
│   ├─ RSI: α_adjusted = 0.05 / 8 = 0.00625
│   └─ 이격도: α_adjusted = 0.05 / 7 ≈ 0.00714
│
├─ 9. 시간대별 분석 (Simple Mode와 동일)
│   ├─ C1이 양봉인 날짜만 필터링
│   └─ calculate_intraday_distribution()
│
├─ 10. 차트 데이터 시각화
│    └─ 패턴 매칭 지점과 차트 데이터
│
└─ 11. 결과 반환
    └─ ComplexAnalysisResult
```

### 연속 봉패턴 분석 (Streak Analysis)

> **⚠️ 수학적 명세 (AI 최적화 방지 가드레일)**: 아래 공식들은 코드 최적화나 리팩토링 시 절대 변경하지 마세요.

**심플 모드:**
- N연속 양봉/음봉 패턴 완성 시점 탐지
- **분석 대상**: 패턴 완성 후 다음 봉(C1, 진입일)
- **C1/C2 예측 확률 (Wilson Score Interval)**:
  - **공식**: $z = 1.96$ (95% 신뢰수준) 기준, 성공 확률 $p$의 상하한 산출
  - **Wilson CI 상한**: $p_{upper} = \frac{p + \frac{z^2}{2n} + z\sqrt{\frac{p(1-p)}{n} + \frac{z^2}{4n^2}}}{1 + \frac{z^2}{n}}$
  - **Wilson CI 하한**: $p_{lower} = \frac{p + \frac{z^2}{2n} - z\sqrt{\frac{p(1-p)}{n} + \frac{z^2}{4n^2}}}{1 + \frac{z^2}{n}}$
  - **제약**: 샘플 수(N) < 10일 경우 'Low Reliability'로 표시
  - **위치**: `backend/strategy/common.py` - `wilson_confidence_interval()`
- 전일 지표 기반 비교 분석 (양봉 케이스 vs 음봉 케이스)
- 뉴욕 시간대별 저점/고점 발생 확률 (1d/3d 타임프레임)
  - C1이 양봉인 날짜의 4시간봉 데이터 분석
  - ✅ EST/EDT 시간대 기준 6개 구간(4시간씩) 확률 계산 (pytz로 DST 자동 처리)
  - 저점과 고점이 **다른** EST/EDT 구간에 있는 날짜만 확률 계산에 포함 (`valid_date_count` 사용)
- 평균 변동성 계산

**복합 모드:**
- 다중 캔들 패턴 매칭 (예: [1,1,1,1,1,-1,-1] = 5양-2음)
- 패턴 완성 시점 탐지
- **분석 대상**: 패턴 완성 후 다음 봉(C1, 진입일)
- **패턴 품질 점수 계산 (Signal Score)**:
  - **목적**: 패턴의 신뢰도 및 품질을 정량화하여 신호 강도 평가
  - **구성 요소**: 가격 지지 강도 (되돌림 비율), 거래량 건강도, 추세 강도 (RSI)
  - **공식**: $Score = Score_{retracement} + Score_{volume} + Score_{rsi}$ (최대 100점)
  - **가격 지지 강도 (되돌림 비율)**: 최대 40점
    - 되돌림 < 30%: 40점 (강력한 지지)
    - 되돌림 < 50%: 30점 (양호한 지지)
    - 되돌림 < 70%: 15점 (보통 지지)
    - 되돌림 >= 70%: 0점 (약한 지지)
    - **되돌림 비율**: $retracement = \frac{rise_{high} - drop_{end}}{rise_{high} - rise_{start}} \times 100$
      - $rise_{high}$: 상승 구간 최고가
      - $drop_{end}$: 조정 구간 종료가
      - $rise_{start}$: 상승 구간 시작가
  - **거래량 건강도 (거래량 비율)**: 최대 30점
    - vol_ratio < 0.7: 30점 (건강한 조정)
    - vol_ratio < 1.0: 20점 (양호한 조정)
    - vol_ratio < 1.5: 10점 (보통 조정)
    - vol_ratio >= 1.5: 0점 (불안한 조정)
    - **거래량 비율**: $vol\_ratio = \frac{vol_{drop}}{vol_{rise}}$
      - $vol_{drop}$: 조정 구간 평균 거래량
      - $vol_{rise}$: 상승 구간 평균 거래량
  - **추세 강도 (RSI)**: 최대 30점
    - RSI > 70: 30점 (강한 추세)
    - RSI > 65: 25점 (양호한 추세)
    - RSI > 60: 15점 (보통 추세)
    - RSI <= 60: 0점 (약한 추세)
  - **신뢰도 판단**:
    - High: $Score \geq 80$
    - Medium: $60 \leq Score < 80$
    - Low: $Score < 60$
  - **위치**: `backend/strategy/common.py` - `calculate_signal_score()`
  - **금지**: 점수 계산 로직, 임계값, 가중치 변경 금지
  - **이유**: 패턴 신호의 객관적 평가 기준 제공
- **RSI/이격도 구간별 승률 분석 (신뢰구간 포함)**:
  - **Wilson Score Interval**: 위 심플 모드 공식 동일 적용
  - **제약**: 샘플 수(N) < 10일 경우 'Low Reliability'로 표시
- **Bonferroni 다중 비교 보정**:
  - **공식**: $\alpha_{adjusted} = \frac{\alpha}{n_{comparisons}}$
    - $\alpha = 0.05$ (원래 유의수준)
    - $n_{comparisons}$: 비교 횟수 (RSI 8구간 = 8, 이격도 7구간 = 7)
  - **적용**: 
    - RSI 구간 분석: 8개 구간 → $\alpha_{adjusted} = 0.05/8 = 0.00625$
    - 이격도 구간 분석: 7개 구간 → $\alpha_{adjusted} = 0.05/7 \approx 0.00714$
  - **위치**: `backend/strategy/common.py` - `analyze_interval_statistics()`
  - **이유**: 다중 비교 시 False Positive 방지 필수
- 뉴욕 시간대별 저점/고점 발생 확률 (1d/3d 타임프레임)
  - C1이 양봉인 날짜의 4시간봉 데이터 분석
  - 저점과 고점이 **다른** EST 구간에 있는 날짜만 확률 계산에 포함
- 차트 데이터 시각화

### 1. 신뢰구간 계산 (Wilson Score Interval)

- **목적**: 소수 샘플 편향 방지
- **공식**: $z = 1.96$ (95% 신뢰수준) 기준, 성공 확률 $p$의 상하한 산출
- **제약**: 샘플 수(N) < 10일 경우 'Low Reliability'로 표시
- **구현 위치**: `backend/strategy/common.py` - `wilson_confidence_interval()` 함수
- **사용처**: 
  - 연속 봉패턴 분석의 C1/C2 예측 확률
  - RSI/이격도 구간별 승률 분석
  - 복합 패턴 분석의 신뢰구간 계산

### 2. 시간대별 분석 (Intraday Distribution)

- **입력**: T시점 패턴 확정 날짜 (C1이 양봉인 날짜)
- **로직**: 해당 날짜의 4시간봉(4h)을 추적하여 뉴욕 시간(EST/EDT) 기준 극값 시간대 매핑
- **수정사항**: 저점/고점이 동일 구간에 발생 시 누락 없이 둘 다 카운트할 것 (WolQuant 개정안)
- **구현 위치**: `backend/strategy/common.py` - `calculate_intraday_distribution()` 함수
- **제약사항**:
  - 1d/3d 타임프레임에서만 동작
  - ✅ EST/EDT 시간대 기준 6개 구간(4시간씩)으로 분할 (pytz로 DST 자동 처리)
  - 저점과 고점이 다른 EST/EDT 구간에 있는 날짜만 확률 계산에 포함 (`valid_date_count` 사용)
  - 변동성 계산은 모든 날짜에 대해 수행

**시각적 다이어그램 (로직 흐름):**
```
패턴 완성 시점 (T)
      ↓
  다음 봉(C1) 확인
      ↓
   C1이 양봉인가?
      ↓ (Yes)
C1 날짜의 4시간봉 추출
      ↓
UTC → EST/EDT 시간대 변환 (pytz 자동 처리)
      ↓
6개 구간(4시간씩) 분할:
  [00:00-04:00 EST/EDT]
  [04:00-08:00 EST/EDT]
  [08:00-12:00 EST/EDT]
  [12:00-16:00 EST/EDT]
  [16:00-20:00 EST/EDT]
  [20:00-24:00 EST/EDT]
      ↓
각 구간별 저점/고점 발생 카운트
      ↓
valid_date_count로 확률 정규화
      ↓
시간대별 확률 계산 완료
```

### 3. 데이터 정규화 (Data Contract)

- **필수**: 모든 분석용 DataFrame은 `DatetimeIndex`를 보유해야 함
- **변환**: 로드 시점에 `open_time`을 인덱스로 강제 설정 및 정규화 수행
- **구현 위치**: `backend/strategy/common.py` - `normalize_indices()` 함수
- **검증**: 인덱스 타입 통일 및 DataFrame 내 존재 여부 확인
- **사용처**: 
  - Simple Mode: `target_cases.index` 정규화
  - Complex Mode: `matched_patterns.index` 정규화
  - C1 날짜 추출 전 인덱스 검증

### 4. 패턴 분석 시점 규칙

- **패턴 완성 시점**: N연속 양봉/음봉의 마지막 봉 또는 복합 패턴의 마지막 봉
- **분석 대상**: 패턴 완성 후 **다음 봉(C1, 진입일)**
- **제약**: 
  - 시간대별 분석은 C1이 양봉인 날짜만 사용
  - C1/C2 예측은 패턴 완성 시점의 다음 봉들을 분석
- **구현 위치**: 
  - `backend/strategy/simple_strategy.py` - `run_simple_analysis()`
  - `backend/strategy/complex_strategy.py` - `_calculate_complex_analysis()`

### 5. 통계 계산 원칙

- **Trimmed Statistics**: 극단값 제외 통계 (변동성 분석 시 사용)
- **Bonferroni Correction**: 다중 비교 시 유의수준 보정
- **Binomial P-value**: 이항 검정을 통한 통계적 유의성 검증
- **구현 위치**: `backend/strategy/common.py`

### 6. 수학적 정의

#### 이격도 (Disparity)
- **공식**: $D = \frac{C - MA_{20}}{MA_{20}} \times 100$
  - $C$: 현재 종가
  - $MA_{20}$: 20일 이동평균선
- **의미**: 현재 가격이 이동평균선으로부터 얼마나 이격되어 있는지 백분율로 표현
- **구현 위치**: `backend/strategy/complex_strategy.py` - `_calculate_indicators()`
- **사용처**: RSI와 함께 구간별 승률 분석에 사용

#### RSI 구간 분석
- **방법**: 
  1. 전일 RSI를 8개 구간으로 분할: `[0, 30, 40, 50, 60, 70, 80, 100]`
  2. 각 구간별로 다음 봉이 양봉일 확률 계산
  3. Binomial test로 통계적 유의성 검증
  4. Wilson Score Interval로 95% 신뢰구간 계산
- **공식**: 
  - Binomial P-value: $p = \sum_{k=x}^{n} \binom{n}{k} p_0^k (1-p_0)^{n-k}$
  - Wilson CI: 위 섹션 1번 참조
- **구현 위치**: `backend/strategy/common.py` - `analyze_interval_statistics()`

#### Bonferroni 보정 (Bonferroni Correction)
- **공식**: $\alpha_{adjusted} = \frac{\alpha}{n_{comparisons}}$
  - $\alpha$: 원래 유의수준 (0.05)
  - $n_{comparisons}$: 비교 횟수 (예: RSI 8구간 = 8, 이격도 7구간 = 7)
- **목적**: 다중 비교 시 1종 오류 증가 방지 (False Positive 방지)
- **적용**: 
  - RSI 구간 분석: 8개 구간 → $\alpha_{adjusted} = 0.05/8 = 0.00625$
  - 이격도 구간 분석: 7개 구간 → $\alpha_{adjusted} = 0.05/7 \approx 0.00714$
- **구현 위치**: `backend/strategy/common.py` - `analyze_interval_statistics()`
- **⚠️ 중요**: 보정 없이는 다중 비교로 인해 우연히 유의한 결과가 나타날 수 있음

### 4. C1 분석 일관성 버그 수정 ✅ **완료** (2025-01-17)

- **문제**: `simple_strategy.py`의 `continuation_count`와 `reversal_count` 계산이 **패턴 완성일(T)** 기준으로 이루어짐
- **올바른 방식**: **C1(T+1)** 기준으로 계산해야 함
- **영향**: Simple Mode의 통계 수치가 부정확하게 계산됨 (버그)
- **수정 내용**:
  - `simple_strategy.py`의 메인 통계 계산 로직 수정 (라인 72-84)
  - `df['next_is_green']` 계산을 메인 통계 계산 전으로 이동
  - `direction`에 따라 C1 기준으로 continuation/reversal 계산
  - `avg_body_pct` 계산도 C1 기준으로 정정
- **수정 코드**:
  ```python
  # direction == "green": C1이 양봉이면 continuation, 음봉이면 reversal
  # direction == "red": C1이 음봉이면 continuation, 양봉이면 reversal
  is_bullish_pattern = (context.direction == "green")
  
  if is_bullish_pattern:
      continuation_count = int(c1_green_mask.sum())
      reversal_count = int(c1_red_mask.sum())
  else:
      continuation_count = int(c1_red_mask.sum())
      reversal_count = int(c1_green_mask.sum())
  ```
- **완료 사항**:
  - ✅ 버그 수정 완료
  - ✅ 문서 업데이트 (`STREAK_ANALYSIS_FLOW.md`)
  - ✅ 변경 이력 추가
